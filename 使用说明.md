# 净帧工坊 0.1 使用说明

## 启动

保持整个 `净帧工坊-免安装版` 目录结构不变，双击 `净帧工坊.exe`。程序不需要安装，视频不会上传；首次启动依赖 Windows 已有的 WebView2 运行时。

## 分发给其他人

直接发送 `净帧工坊-免安装版.zip`。接收者应先把 ZIP 完整解压到普通文件夹，再双击其中的 `净帧工坊.exe`，不要在压缩软件预览窗口里直接运行。FFmpeg、修复引擎、许可证和使用说明都已放在压缩包内，不需要另外安装 Python 或 FFmpeg。当前程序未做商业代码签名，Windows 可能显示“未知发布者”；这不代表程序需要安装，但分发者应同时提供本说明和压缩包的 SHA-256 值供核对。

## 基本流程

1. 导入一个或多个已获授权的视频。
2. 在预览区定位时间，用画笔、矩形或套索添加范围；选区会被限制在视频画面内，重叠的添加选区会自动合并，橡皮擦用于从合并范围中减去。
3. 选择“固定区域”“移动目标”或“透明度变化”范围策略。固定位置但亮度会变化的标记使用“透明度变化”；高清模式会修复整个覆盖范围，不依赖单帧亮度阈值，因此标记变淡时不会突然漏掉。
4. 对移动目标添加关键帧并运行跟踪；时间轴会显示低置信区间，修正关键帧后需重新跟踪。
   如果同一视频既有固定标记又有移动标记：先用“固定区域”框选固定标记，点“保留为固定层，再标记移动目标”，随后框选移动标记并开始跟踪；预览和导出会同时处理两层。
5. 选择修复质量。“高清逐帧（推荐）”使用随软件提供的 LaMa 模型重建复杂纹理；“快速草稿”只用于快速检查选区，复杂背景可能出现模糊色块。点“预览片段”生成当前位置前后约 3 秒的离线修复片段；眼睛按钮切换原片/修复结果。
6. 选择 H.264 或 H.265 与质量值后导出。输出帧率留空时使用源视频平均帧率，也可输入 1–120 fps；“快速转换”复制/删除少量帧，“运动补帧”使用 FFmpeg 运动估计生成中间帧，速度更慢且不是神经网络 AI 补帧。重复导出自动使用 `_clean_02`、`_clean_03`，不会覆盖原文件。
7. “保存进度”会保存选区、关键帧和导出设置为 `.jzf` 项目，稍后可继续编辑。

固定位置的批量视频不需要逐个框选：先在其中一个视频上完成“固定区域”或“透明度变化”选区，再点“将当前选区应用到整批”。程序只会复制到同分辨率、同方向的视频，并把不兼容项目明确跳过；之后可直接使用“批量导出所有就绪视频”。移动目标不会自动套用此功能。

## 资源与模型

- FFmpeg/ffprobe 已放在 `ffmpeg/`，不使用系统 PATH。
- `models/inpainting_lama_2025jan.onnx` 随包提供，固定区域、移动目标和透明度变化的高清模式均可直接离线使用。该模型由 OpenCV 以 Apache-2.0 发布，可重新分发和商业使用。
- 内置双向光流/仿射跟踪器用于移动目标，支持前后传播、多个修正关键帧和低置信区间提示，不需要下载模型。
- ProPainter 与 E2FGVI-HQ 均限制非商业使用，不随包分发，也不会自动下载。商业使用须先获得上游作者授权。
- 基础包默认使用 CPU。检测到 NVIDIA 显卡和驱动后，“处理设备”区域会显示“安装 NVIDIA GPU 加速”；组件约 2.0 GB，安装时至少需要约 4.5 GB 可用空间。下载可暂停和继续，每个文件都会核对大小和 SHA-256；安装到 `engine/gpu-runtime/`，不要求管理员权限，不修改系统 CUDA 或 PATH。只有真实 LaMa 模型通过 CUDA 自检后才会显示“GPU 加速已启用”。
- GPU 组件包括 Apache-2.0 的 LaMa FP32 模型、MIT 的 ONNX Runtime GPU 和 NVIDIA CUDA/cuDNN 运行库。点击安装表示同意 `licenses/GPU-COMPONENT-LICENSES.md` 中列出的 NVIDIA 许可。安装成功后处理仍然完全离线。
- SAM 2 本身许可兼容，但完整 CUDA/PyTorch 跟踪运行时仍会使组件显著变大；当前发行版不把内置跟踪结果冒充为 SAM 2 神经传播结果。

## 输入警告

首版检测到 HDR、10/12-bit 或旋转元数据时会阻止导出，避免静默破坏颜色、方向或同步。可变帧率的 AI 视频会默认启用兼容模式：保留解码得到的全部帧，再按平均帧率或用户指定帧率重建时间轴；建议先生成预览。MP4 文本字幕会映射为 mov_text；图形字幕和其他容器组合可能需要后续版本支持。

## 性能

未安装可选组件时，LaMa 便携引擎使用 CPU；720×1280 局部修复实测约 0.9 帧/秒，因此 3 秒预览可能需要一分钟左右，1 分钟 24 fps 视频约需 25 分钟。RTX 5070 Ti 上的 GPU 浮点模型实测加载约 8.1 秒，首帧（含预热）约 0.92 秒，之后 512×512 推理约 0.07 秒/帧；其他显卡以软件自检和实际任务为准。模型在同一任务中只加载一次，每 4 帧形成一个安全取消单元。Windows DirectML 与量化模型的傅里叶层不兼容，因此 GPU 路径使用经过实测的 CUDA 浮点模型。

## 许可证

第三方模型与 FFmpeg 的来源、再分发和商业使用限制见 `licenses/`。只处理自己拥有或已经获得授权的视频。
