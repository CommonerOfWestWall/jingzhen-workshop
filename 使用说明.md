# 净帧工坊使用说明

这份说明面向直接使用免安装版的普通用户。开发、构建和模型研究请看 [README](README.md)，不需要为了使用软件阅读那些内容。

> 当前版本：0.1 预览版。重要素材请先生成短片预览，并保留原文件。

## 1. 第一次打开

1. 将 `净帧工坊-免安装版.zip` **完整解压**到一个可写目录，例如 `D:\工具\净帧工坊-免安装版`。
2. 双击 `净帧工坊.exe`。
3. 首次打开时，软件会检查 FFmpeg、修复模型、显卡和可选 GPU 组件。检查过程不会上传视频。

不要只复制 `净帧工坊.exe`。它必须和旁边的 `engine`、`models`、`ffmpeg`、`licenses` 文件夹放在一起，否则会出现无法预览或无法导出的错误。

## 2. 界面从哪里开始

| 区域 | 用途 |
| --- | --- |
| 左侧“视频队列” | 导入多个视频、切换当前视频、查看状态和错误 |
| 中间预览区 | 播放、定位、标记选区和对比修复结果 |
| 右侧“范围与导出” | 选择处理策略、跟踪、预览、质量、帧率和导出 |
| 底部时间轴 | 当前帧、关键帧、修正点和低置信区间 |

没有导入视频时，只需要点击 **导入视频**。其余按钮会保持不可用，避免误操作。

## 3. 最快完成一次修复

### 第一步：导入并定位

点击 **导入视频**，选择一个或多个 MP4、MOV、MKV 或 AVI 文件。视频信息读取完成后，队列会显示缩略图、时长、分辨率和帧率。

在中间播放视频，或拖动底部时间轴，找到目标最清楚的一帧。

### 第二步：标记需要修复的范围

工具栏从左到右常用的是：

| 工具 | 适合场景 |
| --- | --- |
| 画笔 | 不规则、小范围补充 |
| 矩形 | 角标、字幕块、规则区域 |
| 套索 | 沿物体轮廓圈选 |
| 橡皮 | 从当前选区减去多余部分 |
| 清空 | 删除当前选区，重新标记 |
| 撤销 / 重做 | 回退或恢复最近一次编辑 |

画笔、矩形和套索添加的内容会合并成**同一个选区**。重叠部分不会被重复处理；如果框多了，使用橡皮、撤销或清空即可。

选区只能在真实视频画面内生效。竖屏视频两侧的黑色留白不是视频内容，不会进入导出结果。

### 第三步：选择范围策略

| 目标表现 | 选择 |
| --- | --- |
| 从头到尾位置不变 | **固定区域** |
| 在画面中移动、缩放或旋转 | **移动目标** |
| 位置基本不变，但会变淡、闪烁或短暂消失 | **透明度变化** |

这三个选项决定“选区随时间怎么变化”，不是三种重复的修复按钮。

### 第四步：跟踪并检查

- **固定区域**：选区会应用到整段视频或指定时间范围，不需要逐帧跟踪。
- **移动目标**：点击 **开始跟踪**，等待软件向前和向后传播掩膜。
- **透明度变化**：软件会保留完整覆盖范围，并按轨迹连续处理，避免目标变淡时选区突然消失。

移动目标跟踪完成后，拖动时间轴检查开头、中间、遮挡处和结尾。时间轴出现低置信提示时：

1. 定位到提示区间内目标最清楚的一帧。
2. 用画笔、矩形或橡皮修正选区。
3. 添加或更新关键帧。
4. 点击 **更新跟踪**。软件只重算受影响的时间区间。

### 第五步：先看预览

点击 **生成预览**。软件会修复当前位置前后约 3 秒，并在完成后启用对比按钮。

- 点击眼睛按钮，在原片和修复结果之间切换。
- 边缘露出目标：适当增加膨胀。
- 边缘过硬：少量增加羽化。
- 出现大片不自然内容：缩小选区，或换到背景更清晰的关键帧重新标记。

默认膨胀和羽化都比较保守。不要为了“一次盖住”把范围扩大到人物五官、手部或细密文字上；修复模型需要足够的周围背景作为参考。

### 第六步：导出

右侧按需要设置：

- **H.264**：兼容性优先，默认推荐。
- **H.265**：文件通常更小，但老设备可能无法播放。
- **质量 CRF**：数字越小质量越高、文件越大；不确定时保留默认值 18。
- **输出帧率**：可输入 1–120 fps。保留原帧率最稳妥；AI 视频节奏不均或需要补帧时再指定 24、25、30、60 等目标值。
- **快速转换**：复制或删除重复帧，速度较快。
- **运动补帧**：通过 FFmpeg 运动估计生成中间帧，画面更顺滑但速度更慢，快速运动和遮挡处可能产生伪影。它不是独立的 AI 补帧模型。

点击 **导出视频**。输出不会覆盖原文件；同名文件会自动使用 `_clean_02`、`_clean_03` 等后缀。

## 4. 同一个视频有固定目标和移动目标

例如左上角有固定标记，右下角还有移动目标：

1. 选择 **固定区域**，标记左上角目标。
2. 点击 **保留为固定层，再标记移动目标**。
3. 标记右下角移动目标。
4. 选择 **移动目标**，开始跟踪并修正关键帧。
5. 生成预览。预览和最终导出会同时处理两层。

## 5. 一批视频使用同一个固定位置

当一批视频的分辨率和方向一致，且目标都在相同位置时：

1. 在其中一个视频上完成固定区域或透明度变化选区。
2. 点击 **将当前选区应用到整批**。
3. 检查队列提示。软件会跳过分辨率或方向不兼容的视频。
4. 抽查几个视频的开头和结尾。
5. 点击 **批量导出所有就绪视频**。

移动目标不能直接套用固定批量选区，因为每个视频的轨迹可能不同。

## 6. 保存进度是做什么的

**保存进度**会创建 `.jzf` 项目文件，记录：

- 视频队列和素材位置；
- 选区、固定层和关键帧修正；
- 跟踪状态、时间范围；
- 预览和导出设置。

下次点击 **打开项目**，可以继续检查和修改，不必重新标记。项目文件不会复制原视频；移动或删除素材后，需要重新找到对应文件。

## 7. 进度、取消和完成状态

一次导出会依次经过：准备素材 → 解码 → 跟踪/修复 → 编码 → 复制音频和字幕 → 校验输出。

帧计数走满只代表“画面帧处理完成”，后面仍可能在写入音频、字幕或校验文件。请以队列中的 **已完成** 和最终输出路径为准。软件会显示当前阶段、已处理帧、速度和预计剩余时间。

点击停止后，软件会在当前安全处理单元结束时取消，避免留下看似完整但实际损坏的视频。取消一个任务不会清空其他队列项。

## 8. NVIDIA GPU 加速

没有独立显卡也能使用，软件会自动采用 CPU 模式，只是速度较慢。

检测到兼容 NVIDIA 显卡时：

1. 在右侧“处理设备”找到 **安装 NVIDIA GPU 加速**。
2. 确认约 2.0 GB 下载量和约 4.5 GiB 可用空间。
3. 开始安装。下载可以暂停，重新打开软件后可继续。
4. 等待真实模型自检完成。只有界面显示 **GPU 加速已启用**，后续任务才会自动使用 CUDA。

组件只安装到软件目录的 `engine\gpu-runtime`，不会修改系统 CUDA 或 PATH。安装失败不影响 CPU 模式。首次安装需要联网，安装成功后视频处理仍然离线。

## 9. AI 生成视频和特殊素材

AI 生成视频常见重复帧、帧间隔不均、局部形变和突然跳变。软件检测到可变帧率时会提示 **AI 视频兼容模式**：保留解码得到的全部帧，再按源平均帧率或用户指定帧率重建时间轴。

建议：

1. 先保留原帧率生成预览，确认音画同步。
2. 只想统一时间轴时使用快速转换。
3. 明显卡顿且运动连续时再尝试运动补帧。
4. 快速运动、手指、头发、文字和遮挡切换处必须检查，因为运动估计算法可能生成错误中间画面。

检测到 HDR、10/12-bit 或复杂旋转元数据时，当前版本会警告或阻止导出，避免静默破坏颜色、方向或同步。

## 10. 常见问题

### 导入后黑屏或尺寸不对

- 确认使用的是完整解压后的免安装目录，不是单独复制出来的 EXE。
- 确认 `ffmpeg`、`engine` 和 `models` 文件夹仍在主程序旁边。
- 关闭程序后重新打开，并重新导入视频。
- 如果仍失败，记录队列中的具体错误和素材参数后提交 Issue；不要只写“未知错误”。

### 为什么“导出视频”不能点击

依次检查右侧步骤：

1. 是否已经导入并选中视频；
2. 是否已经创建有效选区；
3. 移动目标是否完成跟踪；
4. 跟踪结果是否因关键帧修改而变成“需要更新”；
5. 当前任务是否正在预览、导出或取消。

右侧顶部会显示下一步应该做什么。预览是强烈建议的质量检查，但固定区域在选区就绪后也可直接导出。

### 修复结果像模糊块

- 确认选择的是 **高清修复（推荐）**，不是 **快速草稿**。
- 缩小选区，只覆盖真正需要移除的内容。
- 在复杂纹理、人物或文字附近减少膨胀和羽化。
- 对移动目标补充关键帧，避免错误掩膜把正常画面一起修复。
- 先看 3 秒预览；如果背景本身从未露出，任何修复模型都只能推测被遮挡内容。

### 处理很慢

CPU 处理高清视频本来就慢。先用短预览确认范围；有兼容 NVIDIA 显卡时安装 GPU 加速。运行大型游戏、渲染或其他 AI 程序会占用显存并明显降低速度。

### 音频或字幕会丢吗

程序使用显式 FFmpeg 流映射保留兼容音频和文本字幕。MP4 文本字幕可能转换为 `mov_text`。图形字幕、不兼容编码或复杂容器组合仍可能不受支持，软件应给出提示而不是静默忽略。

### 软件会上传视频吗

不会。导入、跟踪、修复、补帧和导出都在本机完成。只有用户主动安装 NVIDIA GPU 组件时会联网下载对应运行库。

## 11. 文件和使用安全

- 始终保留原视频，先对副本或短片测试。
- 不要在导出过程中移动软件目录、原视频或项目文件。
- 分发给其他人时发送完整 `净帧工坊-免安装版.zip`，不要只发送 EXE。
- 可以用包内 `SHA256SUMS.txt` 检查文件是否完整。
- 只处理自己拥有或已经获得授权的视频。

需要报告问题时，请附上：软件版本、Windows 版本、显卡型号、视频分辨率/帧率、所选模式、失败阶段和界面显示的完整错误。不要上传含隐私或未获授权的素材。
